# Feature Implementation Plan

> **作用**: 多步骤任务执行计划模板
> **工具**: Kimi
> **类型**: 工作流 / 计划

## 影响范围

- **作用域**: 当前任务
- **加载时机**: 复杂任务开始时
- **用途**: 结构化多步骤工作，跟踪进度

## 格式说明

```markdown
# Plan: [任务名称]

## Phase 1: 调研与设计
- [ ] 探索现有代码库
- [ ] 识别集成点
- [ ] 设计 API 契约

## Phase 2: 实现
- [ ] 搭建新模块
- [ ] 实现核心逻辑
- [ ] 添加错误处理

## Phase 3: 验证
- [ ] 编写单元测试
- [ ] 运行集成测试
- [ ] 手动验证
```

## 注意事项

- 计划不是填充步骤，而是有意义的、可验证的阶段
- 不要在计划中包含 Agent 无法执行的操作
- 完成后更新 AGENTS.md 记录经验教训
